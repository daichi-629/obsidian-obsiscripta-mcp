# Phase 4, Task 1: Bridge API が 127.0.0.1 のみリッスンを確認

## タスク説明
Bridge API（プラグイン側）が `127.0.0.1` のみにバインドされており、外部アクセス（別ホストからのアクセス）がブロックされていることを確認する。セキュリティポリシー「ローカルホスト専用運用」の遵守を検証する。

## 必須情報
- **テスト前提条件**:
  - Obsidian プラグイン（HTTP サーバー）が起動している
  - Phase 1 が完了している
- **確認コマンド**:
  ```bash
  # ローカルホストからのアクセス（成功すべき）
  curl http://127.0.0.1:3000/bridge/v1/health

  # localhost からのアクセス（127.0.0.1 と同等、成功すべき）
  curl http://localhost:3000/bridge/v1/health

  # 外部ホストからのアクセス（ブロックされるべき）
  # 異なるマシンから、またはドメイン/IP で試行
  curl http://<外部ホスト>:3000/bridge/v1/health
  ```
- **ネットワーク確認**:
  ```bash
  netstat -tuln | grep 3000
  # または
  lsof -i :3000
  ```

## 完了条件
- [ ] `curl http://127.0.0.1:3000/bridge/v1/health` が HTTPステータス 200 を返却する
- [ ] `curl http://localhost:3000/bridge/v1/health` が HTTPステータス 200 を返却する
- [ ] 外部ホストからの接続がタイムアウトまたはコネクション拒否される
- [ ] `netstat` / `lsof` で バインドアドレスが `127.0.0.1` のみであることが確認される
- [ ] プラグイン実装コード（server.ts）で バインドアドレスが hardcoded で `127.0.0.1` に設定されている
